§ 01 — 基础入门

什么是 Docker？

Docker 是一个开源的容器化平台，让开发者可以将应用及其依赖打包成轻量级、可移植的容器，在任何环境中一致运行。

📦

容器化打包

将应用代码、运行时、库文件、配置全部打包为一个独立的容器镜像

🚀

秒级启动

容器启动时间以秒计算，比传统虚拟机快数十倍，大幅提升部署效率

🔄

环境一致性

开发、测试、生产环境完全一致，彻底告别"在我机器上能跑"的困扰

🐳

轻量高效

多个容器共享宿主机内核，无需完整操作系统，开销极低，资源利用率高

🤝

生态丰富

Docker Hub 拥有超过 10 万个官方和社区镜像，几乎涵盖所有常用技术栈

⚙️

DevOps 友好

天然适配 CI/CD 流水线，与 Kubernetes、Swarm 等编排工具无缝集成

§ 02 — 技术对比

Docker vs 传统虚拟机

了解 Docker 容器与传统虚拟机之间的关键差异

对比维度 🖥️ 传统虚拟机 🐳 Docker 容器
架构 每个 VM 运行完整操作系统 + 应用 所有容器共享宿主机 OS 内核
启动时间 数分钟 毫秒 ~ 秒级
资源占用 每个 VM 占用数 GB 磁盘 + 内存 容器仅需 MB 级开销
隔离性 完全硬件级隔离，安全性高 进程级隔离，隔离稍弱但足够
性能 有虚拟化损耗，性能略低 接近原生性能
可移植性 需整体镜像迁移 镜像可一键部署到任何 Docker 环境
规模 单台宿主机通常运行 3~5 个 VM 单台宿主机可运行数十至数百容器
💡 核心区别
虚拟机虚拟化硬件，容器虚拟化操作系统。一台服务器上可以运行数十个容器，但只能运行几个虚拟机——这就是容器如此高效的原因。
§ 03 — 核心概念

Docker 核心概念

掌握 Image（镜像）、Container（容器）、Volume（卷）等关键概念

1

🖼️ Image（镜像）

容器的模板，只读。一个镜像可以创建多个容器。就像一个蛋糕模具——同一个模具可以烤出多个蛋糕。

nginx:alpine
2

📦 Container（容器）

镜像的运行实例，可写、可停止、可删除。每个容器相互隔离，拥有自己的文件系统、网络、进程空间。

docker run nginx
3

📄 Dockerfile

构建镜像的配方文件，描述如何从基础镜像逐步安装依赖、复制文件、配置环境，最终构建出你的自定义镜像。

docker build -t myapp .
4

💾 Volume（数据卷）

用于持久化容器数据。即使容器被删除，卷中的数据依然保留，实现数据与容器分离。

docker run -v /data:/app
5

🔗 Network（网络）

容器间通信的桥梁。支持 bridge、host、overlay 等模式，让容器可以相互访问或与外部网络通信。

docker network create mynet
6

📚 Registry（仓库）

存储和分发镜像的服务。Docker Hub 是最大的公共仓库，也可以搭建私有仓库如 Harbor。

docker push myimage:v1

TUT-06
Docker Compose — 多容器编排
使用 docker-compose.yml 定义和运行多容器应用
+

Docker Compose 用于定义和运行多个容器服务。一个 docker-compose.yml 文件即可启动整套应用栈。

# docker-compose.yml — WordPress + MySQL 示例
version: '3.8'

services:
  db:
    image: mysql:8
    volumes:
      - db_data:/var/lib/mysql
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: secret
      MYSQL_DATABASE: wordpress
      MYSQL_USER: wp_user
      MYSQL_PASSWORD: wp_pass

  wordpress:
    depends_on:
      - db
    image: wordpress:latest
    ports:
      - "8080:80"
    restart: always
    environment:
      WORDPRESS_DB_HOST: db:3306
      WORDPRESS_DB_USER: wp_user
      WORDPRESS_DB_PASSWORD: wp_pass

volumes:
  db_data:
docker-compose up -d
后台启动所有服务
docker-compose ps
查看运行状态
docker-compose logs -f
跟踪指定服务日志
docker-compose down
停止并移除所有服务
docker-compose down -v
同时删除数据卷（⚠️ 数据会丢失）
docker-compose build
重新构建自定义镜像

知识小测验

检验一下你对 Docker 的理解程度！

Q1: 下面哪个命令可以从 Docker Hub 拉取 nginx 镜像？
A docker pull nginx
B docker push nginx
C docker clone nginx
D docker fetch nginx
Q2: Docker 容器与虚拟机的核心区别是什么？
A 容器比虚拟机更安全
B 虚拟机比容器启动更快
C 容器共享宿主机操作系统内核
D 两者没有本质区别
Q3: 以下哪个 Dockerfile 指令用于设置容器启动时执行的命令？
A RUN
B CMD 或 ENTRYPOINT
C FROM
D COPY
Q4: docker run -d --name web -p 8080:80 nginx 中 -p 8080:80 的含义是？
A 容器内端口 8080 映射到 80
B 宿主机端口 8080 映射到容器端口 80
C 同时开放 8080 和 80 两个端口
D 容器使用 8080 和 80 两个端口
Q5: 如何在后台运行一个名为 "app" 的容器，使用 3000 端口？
A docker run -b --name app -p 3000:3000 node
B docker start --name app -p 3000 node
C docker run -d --name app -p 3000:3000 node
D docker exec -d --name app -p 3000 node
